### Название задачи:

## Описание ключевых компонентов

- **Агент на ферме** — локальный сервер на ферме, собирает данные с камер и сенсоров, буферизирует их и отправляет в Kafka. Может выполнять минимальную предобработку (например, агрегацию счётчиков).
- **Центральный сервер** — сервер в ЦОД компании, подписан на Kafka, выполняет расширенную аналитику (включая видеоаналитику с ML-моделями), хранит данные в TimescaleDB/ClickHouse и предоставляет API для мобильных приложений.
- **Агент с локальной аналитикой** — агент, развёрнутый на более мощном сервере фермы с GPU, выполняет локальную видеоаналитику на месте с использованием TensorRT или PyTorch. Отправляет только тревоги и метрики.
- **Центральный сервер (агрегация)** — облегчённый сервер, агрегирует тревоги с агентов, сохраняет их в TimescaleDB и предоставляет API.

### Решение (Основное)

Централизованная архитектура с локальными агентами. Сбор данных с агентов организован через Kafka, который уже используется в компании. Хранение данных выполняется в TimescaleDB или Data Lake с последующей агрегацией в DWH. Централизованный сервер агрегирует информацию, выполняет сложную аналитику, хранение данных, отправляет оповещения и обеспечивает API для внешних систем.

#### Контекстная диаграмма (основное решение)

```plantuml
@startuml
!include https://raw.githubusercontent.com/plantuml-stdlib/C4-PlantUML/master/C4_Context.puml

Person(operator, "Оператор фермы")
Person(engineer, "Инженеры")
Person(mobileApp, "Мобильное приложение")

System(system, "Система мониторинга животных") {
  System_Ext(feeders, "Кормушки и поилки")
  System_Ext(cameras, "Видеокамеры")
  System_Ext(sensors, "Датчики среды")
  System_Ext(kafka, "Kafka", "Шина данных компании")
  System_Ext(tsdb, "TimescaleDB", "Хранилище временных рядов")

  System(edgeAgent, "Агент на ферме")
  System(centralServer, "Центральный сервер")
}

Rel(edgeAgent, operator, "Локальные оповещения")
Rel(edgeAgent, kafka, "Публикация данных с камер и сенсоров")
Rel(kafka, centralServer, "Получение данных")
Rel(centralServer, tsdb, "Сохранение метрик и видеоаналитики")
Rel(centralServer, operator, "Оповещения через мобильное/web приложение")
Rel(centralServer, mobileApp, "API доступ к данным")
Rel(engineer, centralServer, "Управление оборудованием")
Rel(edgeAgent, feeders, "Управление кормушками")
Rel(edgeAgent, cameras, "Получение видеопотока")
Rel(edgeAgent, sensors, "Сбор данных с сенсоров")
@enduml
```

### Альтернативы

#### Альтернативное решение:

Распределённая аналитика с локальной предобработкой. Агент выполняет локальную видеоаналитику, отправляет агрегированные события в Kafka с последующим сохранением в TimescaleDB/DWH. Подходит для ферм с плохим интернетом.

#### Контекстная диаграмма (альтернативное решение)

```plantuml
@startuml
!include https://raw.githubusercontent.com/plantuml-stdlib/C4-PlantUML/master/C4_Context.puml

Person(operator, "Оператор фермы")
Person(engineer, "Инженеры")
Person(mobileApp, "Мобильное приложение")

System(system, "Система мониторинга животных") {
  System_Ext(feeders, "Кормушки и поилки")
  System_Ext(cameras, "Видеокамеры")
  System_Ext(sensors, "Датчики среды")
  System_Ext(kafka, "Kafka", "Шина данных компании")
  System_Ext(tsdb, "TimescaleDB", "Хранилище временных рядов")

  System(edgeAgent, "Агент с локальной аналитикой")
  System(centralServer, "Центральный сервер (агрегация)")
}

Rel(edgeAgent, operator, "Локальные оповещения и аналитика")
Rel(edgeAgent, kafka, "Публикация метрик и тревожных событий")
Rel(kafka, centralServer, "Получение событий")
Rel(centralServer, tsdb, "Сохранение агрегированных данных")
Rel(centralServer, operator, "Оповещения через мобильное/web приложение")
Rel(centralServer, mobileApp, "API для внешних систем")
Rel(engineer, edgeAgent, "Локальное управление оборудованием")
Rel(edgeAgent, feeders, "Управление кормушками")
Rel(edgeAgent, cameras, "Локальная видеоаналитика")
Rel(edgeAgent, sensors, "Сбор и анализ данных с сенсоров")
@enduml
```

### Недостатки, ограничения, риски

**Основное решение:**

- Зависимость от качества связи и Kafka;
- Плюс — переиспользуем корпоративную инфраструктуру;

**Альтернативное решение:**

- Увеличенная стоимость edge;
- Плюс — автономность ферм без интернет;
- Все данные всё равно синхронизируются с Kafka для единой аналитики компании.
